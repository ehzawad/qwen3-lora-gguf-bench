# scripts/

Flat by design (each script resolves the repo root as its parent dir; a subdir
split was deliberately avoided to keep reproduction paths stable). Grouped here
by purpose. Everything is driven by [`../run.sh`](../run.sh); the two multi-agent
workflow definitions live in [`../workflows/`](../workflows).

### Pipeline (base + LoRA → GGUF Q6_K)
| Script | Role |
|---|---|
| `01_download.py` | download pinned base + LoRA adapter, verify adapter SHA |
| `merge.py` | PEFT merge (verified: bit-exact, adapter baked in) |
| `03_build_llamacpp.sh` | build pinned llama.cpp with CUDA (arch 86) |
| `convert.sh` | merged HF → bf16 GGUF (`--outtype bf16`) |
| `quantize.sh` | bf16 GGUF → Q6_K (also used to make Q4_K_M/Q5_K_M/Q8_0) |
| `verify_artifacts.py` | GGUF metadata / tied-head / vocab / template checks |
| `make_corpus.py` | seeded short-chat prompt corpus |
| `capture_env.py` | sanitized hardware/env manifest per run |

### Serving benchmarks (concurrency, precision, engines, load)
| Script | Study | Role |
|---|---|---|
| `benchmark.py` | Work A / E1 | closed-loop concurrency benchmark (starts its own server) |
| `sweep_concurrency.sh` | E1 | C=1..128 sweep + VRAM/slot + CPU telemetry |
| `precision_sweep.sh` | E4 | llama-bench weight-quant × KV-dtype × flash-attn |
| `engine_compare.sh` + `bench_external.py` | E5 | llama.cpp vs vLLM (vLLM crashed → void), external-server harness |
| `locustfile.py` + `locust_run.sh` | E2/E3 | synthetic finite-user (think-time) load |

### Context window & accuracy (this round)
| Script | Study | Role |
|---|---|---|
| `prepare_data.py` | — | download wikitext-2 (perplexity) + ultrachat (multi-turn) |
| `context_sweep.sh` | E6 | decode-vs-KV-depth, per-slot VRAM, fit/no-fit frontier (f16 & q8_0) |
| `perplexity.sh` | E7 | llama-perplexity per quant → ΔPPL vs bf16 (accuracy↔speed) |
| `composite_sessions.py` | — | build 8k/16k/32k **composite** sessions from real ultrachat transcripts |
| `chat.py` | — | interactive multi-turn chatbot CLI (prefix caching, prints TTFT/cached) |
| `multiturn_replay.py` | — | single-user multi-turn replay (cache on/off A/B) |
| `mt_concurrent.py` | — | **N concurrent multi-turn users** (slot-affinity) at a target context |

### Analysis
| Script | Role |
|---|---|
| `svd_extract.py`, `compare_extracted.py` | Work B: SVD LoRA-extraction probe |
| `report.py` | aggregate benchmark JSON → summary table |
| `check_repo_hygiene.sh` | pre-commit artifact/size/gitlink guard |

Datasets land in `../data/` (git-ignored; regenerated by `prepare_data.py`).
Model weights, GGUFs, `vendor/`, and `logs/` are git-ignored; only scripts,
config, prompts, results JSON/CSV, and reports are committed.
